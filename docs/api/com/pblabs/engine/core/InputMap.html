Classpublic class InputMap
InheritanceInputMap Object
Implements ISerializable

This class wraps the InputManager to allow for better control over input events. It also provides a mechanism for creating custom keybindings and serializing those bindings.

The way this is accomplished is by abstracting input events by giving them names, and then mapping those names to specific keys. So, registering a method to be called when a key is pressed is done by registering that method with the name representing the event, then registering the specific key with the name.

mapActionToHandler()method 
public function mapActionToHandler(actionName:String, handler:Function):void

Binds an input action to a handler callback. When the specified input action is triggered, the handler function will be called. If it is a press event, the function will be passed 1 as its only parameter. If it is a release event, the function will be passed 0 as its only parameter. If it is an analog event then the analog value will be passed. In the case of mouse movement (currently the only analog event) the value will be the amount the mouse moved on the specific axis. Input actions are mapped to specific key or mouse inputs in SetKeyMapping(). Note: If you are just binding a specific key directly to a specific handler function (with no special key remapping options), then using MapKeyToHandler() will save you a step. Use the MapKeyToAction() and MapActionToHandler() pair if you want the flexibility of an abstracted re-usable component or easier key re-binding.

Parameters

actionName:String — The name to give this binding.
 
handler:Function — The function to call when the input event defined by actionName is triggered.

mapKeyToHandler()method 
public function mapKeyToHandler(key:InputKey, handler:Function):void

Maps a specific key to a handler callback. When the specified key or mouse input is triggered, the handler function will be called. If it is a press event, the function will be passed 1 as its only parameter. If it is a release event, the function will be passed 0 as its only parameter. If it is an analog event then the analog value will be passed. In the case of mouse movement (currently the only analog event) the value will be the amount the mouse moved on the specific axis. Note: If you need special key remapping options, or are building an abstracted re-usable component, you may want the flexibilty offered by using MapKeyToAction() and MapKeyToHandler(), as this lets you separate specific keys from specific callbacks, as they can be bound later in game configuration files (such as .pbelevel files).

Parameters

key:InputKey — The key that will trigger the handler. This should be one of the constants defined in the InputKey class.
 
handler:Function — The function to call when the input event is triggered.
